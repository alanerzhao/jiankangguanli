# Acceptance

这些条目定义了当前版本最少必须成立的行为。

## 本地模式

1. 在没有有效 Supabase 配置时，页面可以正常打开、填写、保存和刷新。
2. 新增一条记录后，最近记录表、顶部摘要、复盘卡片和趋势图都会更新。
3. 对同一天再次保存，会覆盖同日期旧记录，而不是新增重复日期。
4. 清空本地数据后，界面回到空状态，并保留“不会自动删除云端数据”的提示语义。
5. 导出数据必须包含 `goals`、`records`、`cloud` 和 `exportedAt`。
6. 导入合法 JSON 后，记录和目标会被还原。

## 复盘与趋势

7. 复盘周期支持 `7d`、`30d`、`month`、`year`。
8. 趋势图指标支持 `water`、`sleep`、`steps`、`calories`、`weight`。
9. 没有足够数据时，趋势图和复盘区应显示空状态，而不是报错。

## 云同步

10. 配置了 Supabase 但尚未完成授权时，页面不应阻断本地记录。
11. 授权成功后，页面加载时会尝试拉取云端记录和目标。
12. 本地新增记录、修改目标后，若已有云端会话，应自动写入云端。

## 交付检查

每次改动至少跑下面这些命令：

```bash
npm run check
```

发布前建议再手动检查：

```text
保存记录 -> 切换趋势指标 -> 切换复盘周期 -> 刷新页面
```
